# Checklist Gap Matrix — TOAD vs `agent_teams_hardening_checklist_final_v2.md`

Last audit: 2026-04-30 (after the plan-before-code slice).

This document is the single source of truth for "where is TOAD against the v2 hardening checklist?" The full checklist lives at `docs/AGENT_TEAMS_HARDENING_CHECKLIST.md`. This file is updated at the end of every slice that touches enforcement scope.

## Severity legend

- **REAL** — implemented, tested, reachable from the API surface.
- **PARTIAL** — scaffolding present but missing pieces the checklist calls out.
- **MISSING** — not implemented at all in TOAD.
- **OUT-OF-SCOPE-NOW** — intentionally deferred per project priorities (e.g. UI work).

## Recommended slice order (anchored to the checklist's own priority)

The checklist's stated order is `state machine > tool authority > session tracking > diff tracking > CI gates > diagnostics > UI polish`. Mapping to TOAD's current state and the gaps above:

1. ✅ **Deterministic state machine (§3)** — done. 10-state lifecycle, transition validator, `from`/`reason` recorded per checklist.
2. ✅ **Role authority (§5 + §26)** — done (with permissive default for backward compat). Allowlist enforced in `LocalToolFacade.execute`; self-review prevention in `#reviewDecide`.
3. ✅ **Test artifacts + CI gates (§6 + §18)** — done. Validation config on TeamConfig; `validation_run` tool; `task.validations[]` projection; `testing → merge_ready` blocked without a passing test verdict.
4. ✅ **Plan-before-code (§2)** — done. Three plan tools (propose/approve/reject), `task.plan` projection, ready→planned gate, self-approval prevention.
5. ✅ **Diagnostics (§25)** — done. `diagnostics_run` read-only MCP tool re-runs the four enforcement layers (state machine, role authority, plan/CI gates) and probes provider availability + DB persistence.
6. ✅ **Per-transition role guards (§3 × §5)** — done. `merge_ready → done` restricted to lead/human; `rejected → backlog` and `blocked → *` restricted to architect/lead/human. Architect gained `task_update` access. Wired through `actor.role` in `#taskUpdate`.
7. ✅ **`tool_call_denied` event emission (§26)** — done. Denied tool calls now emit a runtime event before re-throwing. Best-effort (broken event log doesn't mask the role-authority error).
8. ✅ **Worktree-per-task slice 1 (§8)** — done. Creation half: `WorktreeManager.createForTask` runs `git worktree add` on `ready → planned`; projection captures `task.worktree`. Best-effort with skip variants for non-git workspaces.
9. ✅ **Worktree-per-task slice 2 (§8)** — done. `agent_launch` enforces cwd against `task.worktree.path` (auto-set on omit, reject on conflict). `taskId` passed through to launch input.
10. ✅ **Worktree-per-task slice 3 (§8)** — done. `WorktreeManager.removeForTask` runs `git worktree remove --force` on `merge_ready → done`. Branch preserved. `rejected` doesn't auto-remove (operator triages WIP).
11. ✅ **Diff tracking (§7 finished)** — done. `computeDiff` runs `git diff baseRef..HEAD` inside the worktree; `review_request` auto-attaches `diff` + `files` when caller omits them. Operator override always wins.
12. ✅ **Merge workflow (§19 slice 1)** — done. `checkForConflicts` runs `git merge --no-commit --no-ff` + `--abort` to verify the task branch can be merged into `baseRef` cleanly. Conflict or error blocks `merge_ready → done`.
13. ✅ **Scope-drift detection (§13 partial)** — done. `task.review.scopeDrift[]` lists files outside `task.plan.filesExpectedToChange` after diff is captured. Empty plan list = no flagging.
14. ✅ **No-op diff detector (§13 partial)** — done. `task.review.noOpDiff` flags review_request where the orchestrator computed an empty diff (zero files). Reviewer-informational.
15. ✅ **Repeated test-failure detector (§13 partial)** — done. `task.consecutiveTestFailures` derived from `task.validations` trailing streak; `task.repeatedTestFailures` flips at ≥3.
16. ✅ **Worktree slice 4 (§8)** — done. `task_create` accepts explicit `baseRef` + `baseBranch`. `WorktreeManager.createForTask` forwards the override, skipping `rev-parse HEAD`. §8 now fully **REAL**.
17. ✅ **Session→task pinning (§11 slice 1)** — done. `runtime_instances.task_id` column persists the link. `agent_launch.taskId` flows facade → supervisor → registry. Soft migration handles existing dbs.
18. **Merge slice 2 (§19) — NEXT.** Actually perform the integration commit on `baseBranch`. Now unblocked by the baseRef + baseBranch capture.
14. **Merge slice 2 (§19).** Actually perform the integration commit on `baseBranch` (currently the gate only verifies feasibility — operator does the real merge).
11. Failure detection (§13), WIP limits (§9), dependency enforcement (§10), notifications, knowledge propagation. Smaller incremental slices once the spine is in place.
