# Checklist Gap Matrix — TOAD vs `agent_teams_hardening_checklist_final_v2.md`

Last audit: 2026-05-01 (after the Level 1/2/3 smoke test re-evaluation).

This document is the single source of truth for "where is TOAD against the v2 hardening checklist?" The full checklist lives at `docs/AGENT_TEAMS_HARDENING_CHECKLIST.md`. This file is updated at the end of every slice that touches enforcement scope.

## Architectural context (2026-05-01 re-evaluation)

After running an end-to-end smoke against a real Claude CLI, three things became clear that were not obvious from the unit-test view:

1. **TOAD's orchestrator is genuinely complete** for the parts of the v2 checklist that are about enforcement (state machine, role authority, plan / CI / merge gates, scope-drift / no-op-diff detectors, audit export). The orchestrator works end-to-end against real git operations.
2. **The agent-side MCP tool surface is wired and real-agent verified.** `LocalToadRuntime.launchAgent()` writes a per-runtime MCP config and appends `--mcp-config <path>` for Claude commands when TOAD has a persistent DB and project root. The stdio MCP server opens the same `TOAD_DB_PATH` / `TOAD_PROJECT_CWD` and receives explicit actor role env, so a launched agent can call `task_update`, `review_request`, `message_send`, etc. against the live orchestrator state. Level-3 smoke confirmed a real Claude agent discovered `toad-local`, called `task_comment`, received the orchestrator response, and mutated live task state.
3. **TOAD has correctly diverged from the upstream `claude_agent_teams_ui-main` reference** wherever the checklist demanded it (state machine, gates, detectors). The places we still need to align with upstream are:
    - **MCP tool surface for agents.** Upstream passes `--mcp-config <path>` to claude on launch, with the config pointing at a server that exposes the team-coordination tools. TOAD now mirrors that core launch shape while keeping its own orchestrator-managed worktree/gate model.
    - **Bootstrap prompt + inbox-file delivery model** (large, optional). Upstream uses a `--team-bootstrap-spec` JSON file and inbox JSON files for ongoing teammate-to-teammate messaging instead of stdin push. Adopting this would be a multi-slice architectural shift. (≈ 5–10 slices — see Option 3 in `HANDOFF-NEXT-AGENT.md`.)

## Severity legend

- **PARTIAL** — scaffolding present but missing pieces the checklist calls out.
- **MISSING** — not implemented at all in TOAD.
- **OUT-OF-SCOPE-NOW** — intentionally deferred per project priorities (e.g. UI work).
